# AML RBAC starter pack

This starter pack contains a PostgreSQL schema and seed data for AML platform RBAC plus a Keycloak realm import skeleton.

## Folder structure
- `sql/01_extensions.sql` - required PostgreSQL extensions
- `sql/02_schema.sql` - users, tenants, roles, permissions, alerts, cases, ACL, audit tables
- `sql/03_seed_rbac.sql` - starter roles, permissions, and mappings
- `sql/04_rls.sql` - row-level security policies
- `sql/05_helper_functions.sql` - helper functions for permission checks and audit logging
- `sql/06_example_bootstrap.sql` - example tenant and users
- `sql/07_runtime_usage.sql` - per-request session context example
- `keycloak/realm-overwatch-aml.json` - importable realm skeleton
- `keycloak/composite-role-plan.md` - post-import composite role setup notes

## Apply order
1. `01_extensions.sql`
2. `02_schema.sql`
3. `03_seed_rbac.sql`
4. `04_rls.sql`
5. `05_helper_functions.sql`
6. `06_example_bootstrap.sql`

## App runtime contract
At the start of each request, set:
- `app.current_tenant`
- `app.current_user_id`

Then rely on RLS plus explicit permission checks through `app.user_has_permission(...)`.
